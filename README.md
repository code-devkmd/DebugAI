# DebugAI
[![License](https://img.shields.io/badge/license-MIT-green.svg)](LICENSE)
[![Status](https://img.shields.io/badge/status-active-success.svg)]()
[![Made with](https://img.shields.io/badge/made%20with-JavaScript-blue.svg)]()

DebugAI is an AI-powered debugging assistant that helps developers quickly identify, understand, and fix errors in their code. It uses Groq’s LLM API (Llama 3) to analyze stack traces, runtime errors, and buggy code, and returns clear root cause explanations along with actionable fixes.

---

## Features

- AI-powered error analysis using Groq API (Llama 3)
- Supports stack traces, logs, and raw code input
- Root cause identification with clear explanations
- Direct, actionable code fixes generated by AI
- Clean and responsive UI built with React and Tailwind CSS
- Markdown rendering for formatted responses (code blocks, inline code, highlights)

---

## Tech Stack

- **Frontend:** Next.js / React
- **Styling:** Tailwind CSS
- **AI Model:** Groq API (Llama 3 70B / 8B)
- **Rendering:** Custom Markdown Parser

## Installation

### 1. Clone the repository

```bash
git clone https://github.com/code-devkmd/DebugAI.git
cd DebugAI
```

### 2. Install dependencies
```bash
npm install
```

### 3. Set up environment variables

Create a .env file in the root directory:
```bash
 GROQ_API_KEY=your_api_key_here
```

Do NOT hardcode API keys inside the source code.

### 4. Start the development server
```bash
npm start
```

## Usage
 1. Enter the programming language or framework (optional)
 2. Paste your error log or buggy code
 3. Click Analyze Error
 4. Receive:
     - Root cause explanation
     - Fixed code solution

## Project Structure
```bash
 app/
 ├── page.js              # Main UI logic
 ├── api/
 │    └── analyze/route.js # Backend API route (Groq call)
 ├── components/          # Reusable UI components
 ├── globals.css          # Tailwind global styles
```

## Security Note

Do not expose API keys in client-side code. Use environment variables or a backend proxy in production.
